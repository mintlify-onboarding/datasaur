After [signing in](https://app.datasaur.ai/sign-in), you will be automatically directed to your personal workspace. You can see the Project shortcuts and the list of Projects that you are working on.

![](/images/datasaur/getting-started/First%20Page%20Datasaur.png)

Creating a Project can be done by clicking on the **Create project** button, or on one of the **Project Template** shortcuts. In this article, we will walk through creating a custom Project and [the next article describes each of the Project Templates](/datasaur/overview/task-type/project-templates). If you would like a tutorial on creating a project for [token](https://www.youtube.com/watch?v=T%5FG7wWx7LRg), [audio](https://www.youtube.com/watch?v=9IrY-xcta7w), [OCR](https://www.youtube.com/watch?v=YsObDLKsewY), or document based Project please watch their corresponding Youtube videos.

## Project Creation Wizard

The Project Creation Wizard is a tool for creating custom Projects. It has three basic steps: **add data**, **preview data**, and **labeler tasks**. (A fourth step is available for team admins to **assign labelers**).

You are able to upload multiple files, **but all files in a Project must be the same file format.**

Uploading the data can be done in two ways: **drag and drop** and **browsing files from your hard drive**.

If you are interested in creating Projects via API, you can find a [full article here](/datasaur/api/create-new-project).

_Note: the maximum file size allowed is 50 MB._

### Step 2: Preview the uploaded data ([Video Tutorial](https://www.loom.com/share/efae3c061e20499f907930f2bc11f3dd))

This step allows you to preview what your data will look like.

![](/images/datasaur/getting-started)

The **Number of rows displayed per page** setting determines how many rows should be displayed on one page. Choosing _All rows_ will allow infinite scrolling through all the data.

The **Expand Media** setting allows you to choose the media resolution that will be displayed on the page. Higher resolutions will allow you to view the media in greater detail, but will take longer to load.

The **Enable markdown parsing** allows you to parse markdown in row-based project. We recommend preprocessing your file with Markdown syntax before uploading it to Datasaur.

You can also edit the header and hide columns by right-clicking the header.

### Step 3: Labeler Tasks ([Video Tutorial](https://www.loom.com/share/f23d1d1dcabc41ffb192d1c334452d58))

In this last step, you must choose whether you want to label individual tokens or answer questions about the text.

There are three task types that you can choose: **token-based**, **row-based**, and **document-based**.

* [**Token-based**](https://github.com/vulcan-ai/gitbook/blob/datasaur/getting-started/creating-a-project/broken-reference/README.md) allows you to label tokens in a text document.  
   * If you would like a tutorial on token-based Projects, please watch the [corresponding YouTube videos.](https://youtu.be/T%5FG7wWx7LRg)

* [**Row-based**](https://github.com/vulcan-ai/gitbook/blob/datasaur/getting-started/creating-a-project/broken-reference/README.md) allows you to label data in tables on a row-by-row basis.  
   * If you would like a tutorial on row-based Projects, please watch the [corresponding YouTube videos.](https://youtu.be/KDRHJ7JVuDk)

* [**Document-based**](https://github.com/vulcan-ai/gitbook/blob/datasaur/getting-started/creating-a-project/broken-reference/README.md) allows you to label entire documents at a time.

![](/images/datasaur/getting-started/image%20(8)%20(1)%20(2)%20(1).png)

You can create or upload label sets
